Can you use iFrames instead of PHP include()?

Sure.
How easy is it? In my experience, I've found it pretty unrelated.

I started thinking about this problem when trying to add some PHP to this very site as a means of keeping headers, navigation, footers, and just about anything I wanted to keep consistent through out the site. But since my site is hosted by Github, and since Github does not support PHP (as far as I could tell), I wanted to see if there was an alternative solution.

Issues I've encountered so far:

  • Styling is a bit of a mess if you're adding on to an existing site.
  • Scrollbar decides to stick around where it's beem unwanterd.
  • allow-top-navigation hasn't been working. (Turns out all I needed to do was to add target='_top' to the link)
  • Refreshing the page causes to navigate back to the index page, i.e. loses the inner iframe content.
